# Familiada – szczegółowy przepływ i wymagania

## Layout

| Element | Rola |
|---------|------|
| **Ekran TV (duży)** | Wyświetla tabelę z pytaniami i odpowiedziami (odkrywanymi przez admina). |
| **Telefon admina** | Kieruje grą: przyznaje punkty, odsłania odpowiedzi, steruje rundą. |
| **Gracze** | Nie logują się, nie widzą aplikacji quizowej. Używają tylko przycisków „kto szybciej". |

## Przyciski graczy – dwie opcje

| Opcja | Opis |
|-------|------|
| **A) Sprzętowe (offline)** | Fizyczne przyciski typu buzzer – działają bez internetu, bez aplikacji. |
| **B) W aplikacji (telefon)** | Ekran podzielony na dwa pola: **niebieskie** i **czerwone** (drużyny). Kto pierwszy naciśnie – odpowiada i ma szansę na punkty. Opcja software'owa w telefonie. |

## Przepływ gry

1. Ekran TV pokazuje tabelę z odpowiedziami (np. 6 rzędów, każde odsłaniane po kolei).
2. Admin na telefonie: „Kto pierwszy?" → przyciski graczy (buzzer lub pola niebieskie/czerwone) są aktywne.
3. Pierwszy naciśnięty – ta drużyna odpowiada. Admin przyznaje punkty / odsłania następną odpowiedź.
4. Admin steruje: odsłanianie, punkty, kolejne pytanie.
